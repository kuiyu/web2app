## 一、产品定位与核心价值

### 1.1 产品定位

**WebAppLauncher 是一个面向企业级场景的 Web 应用桌面容器。**

它不是浏览器，而是一个**可配置、可管控、可扩展的桌面外壳**，将多个 Web 应用以“本地软件”的形式统一交付和管理。

### 1.2 核心价值主张

> **“让 Web 应用拥有桌面软件的体验，让桌面管理拥有 Web 应用的敏捷。”**

| 维度       | 传统方式                 | WebAppLauncher          |
| :--------- | :----------------------- | :---------------------- |
| 入口管理   | 分散的网址、收藏夹       | 统一桌面应用，菜单切换  |
| 新应用上线 | 发通知、培训、配置浏览器 | 改配置文件，零代码上线  |
| 安全管控   | 依赖浏览器策略，难以统一 | 统一禁用右键、F12、弹窗 |
| 用户体验   | 浏览器工具栏干扰         | 无干扰的全屏沉浸体验    |
| 运维成本   | 逐个系统维护             | 集中配置，一键检查      |

### 3.2 应用容器模块

#### 3.2.1 Web 应用能力边界

| 能力项                | 支持情况   | 说明                   |
| :-------------------- | :--------- | :--------------------- |
| 静态 HTML/JS/CSS      | ✅ 完全支持 | 本地文件加载           |
| 跨域请求              | ✅ 支持     | 遵循 WebView2 安全策略 |
| localStorage / Cookie | ✅ 支持     | 与普通浏览器行为一致   |
| 后端 API 调用         | ✅ 支持     | 通过 HTTP/HTTPS 访问   |
| 本地文件系统访问      | ❌ 默认禁止 | 需通过扩展机制授权     |
| 系统 API 调用         | ❌ 默认禁止 | 需通过扩展机制授权     |

#### 3.2.2 应用切换与生命周期

| 行为         | 说明                                                   |
| :----------- | :----------------------------------------------------- |
| 切换方式     | 菜单点击切换，无需重启应用                             |
| 状态保留     | 切换后原应用状态**默认销毁**，重新加载时从初始状态开始 |
| 并发运行     | 当前版本**不支持**多应用同时运行，一次仅运行一个应用   |
| 生命周期事件 | 支持应用启动、切换、关闭等事件钩子（预留扩展）         |

> **设计考虑**：单应用运行模式降低了资源占用和状态管理的复杂性，适合企业内部工具场景。如需多应用并行，后续版本可考虑标签页模式。

### 3.3 配置管理模块

#### 3.3.1 完整配置结构

json

```
{
  "WebAppSettings": {
    "CurrentApp": "app1",
    "Apps": {
      "app1": {
        "Name": "示例应用1",
        "Path": "apps/app1/index.html",
        "Title": "我的Web应用1",
        "Run": "",
        "RunBefore": "",
        "RunAfter": "",
        "RunTimeout": 30,
        "AllowedDomains": [],
        "Permissions": ["webview2-default"]
      }
    },
    "Global": {
      "EnableDevTools": false,
      "EnableContextMenu": false,
      "DefaultPermissions": ["webview2-default"]
    }
  },
  "WindowSettings": {
    "Width": 1200,
    "Height": 800,
    "StartPosition": "CenterScreen",
    "DisableContextMenu": true,
    "MinWidth": 800,
    "MinHeight": 600,
    "Maximized": false
  },
  "Logging": {
    "Enabled": true,
    "Level": "Info",
    "Path": "logs/app.log",
    "MaxSizeMB": 10
  }
}
```



#### 3.3.2 新增配置字段说明

| 字段             | 类型   | 说明                                           |
| :--------------- | :----- | :--------------------------------------------- |
| `Run`            | string | 应用启动时**同时运行**的外部程序路径（可选）   |
| `RunBefore`      | string | 应用加载**前**执行的外部程序（如启动本地服务） |
| `RunAfter`       | string | 应用关闭**后**执行的外部程序（如清理资源）     |
| `RunTimeout`     | int    | 外部程序等待超时时间（秒），默认 30            |
| `AllowedDomains` | array  | 允许访问的域名白名单（空表示无限制）           |
| `Permissions`    | array  | 应用权限列表，控制本地能力访问                 |
| `EnableDevTools` | bool   | 全局/应用级开发者工具开关                      |
| `Logging`        | object | 日志配置，支持按大小滚动                       |

#### 3.3.3 Run 扩展机制说明

> ⚠️ **安全提示**：`Run` 系列字段涉及外部程序启动，建议仅在受信任环境中使用，并对启动程序路径进行校验。

**执行时序**：

text

```
启动应用
    ↓
RunBefore (可选，等待完成或超时)
    ↓
加载 Web 内容
    ↓
Run (可选，并行运行)
    ↓
用户使用
    ↓
切换/关闭应用
    ↓
RunAfter (可选，清理)
```



**使用示例**：

json

```
{
  "app1": {
    "Name": "本地开发面板",
    "Path": "apps/app1/index.html",
    "RunBefore": "start-local-server.bat",
    "RunAfter": "stop-local-server.bat",
    "RunTimeout": 10
  }
}
```



#### 3.3.4 热加载机制

| 配置变更类型      | 是否需要重启 | 生效方式           |
| :---------------- | :----------- | :----------------- |
| 修改 `CurrentApp` | ❌ 不需要     | 菜单点击"重新加载" |
| 修改应用 `Path`   | ❌ 不需要     | 下次启动该应用时   |
| 修改窗口尺寸      | ✅ 需要       | 重启应用           |
| 修改 `Run` 系列   | ❌ 不需要     | 下次启动该应用时   |
| 新增/删除应用     | ❌ 不需要     | 菜单自动刷新       |

> **热加载操作路径**：工具 → 重新加载配置

### 3.4 安全管理模块

#### 3.4.1 安全策略矩阵

| 安全项         | 实现方式                                   | 可配置性      |
| :------------- | :----------------------------------------- | :------------ |
| 右键菜单禁用   | WebView2 设置 + JavaScript 注入 + 窗体事件 | ✅ 全局/应用级 |
| 开发者工具禁用 | `AreDevToolsEnabled = false`               | ✅ 全局/应用级 |
| 新窗口弹出拦截 | `NewWindowRequested` 事件处理              | ✅ 固定开启    |
| 外部程序启动   | `Run` 字段控制                             | ✅ 应用级      |
| 域名访问控制   | `AllowedDomains` 白名单                    | ✅ 应用级      |
| 本地权限控制   | `Permissions` 字段                         | ✅ 应用级      |

#### 3.4.2 权限模型（预留扩展）

json

```
{
  "Permissions": [
    "webview2-default",    // 默认权限：正常网页功能
    "allow-local-file",    // 允许访问本地文件（需授权）
    "allow-system-api"     // 允许调用系统 API（需授权）
  ]
}
```



> 当前版本仅支持 `webview2-default`，扩展权限为后续版本预留。

### 3.6 运维工具模块

#### 3.6.1 日志规范

| 日志级别 | 使用场景                 | 示例                                   |
| :------- | :----------------------- | :------------------------------------- |
| Error    | 应用崩溃、配置加载失败   | `Failed to load app config: {error}`   |

| 故障类型            | 恢复策略                               |
| :------------------ | :------------------------------------- |
| WebView2 初始化失败 | 自动重试 3 次，提示用户安装/修复运行时 |
| 应用加载失败        | 显示错误页面，提供"重新加载"按钮       |
| 配置文件损坏        | 尝试恢复备份，失败则使用默认配置       |
| 外部程序启动失败    | 记录日志，继续加载 Web 内容（不阻断）  |

## 七、安全与合规说明

### 7.1 已知安全边界

- Web 应用运行在 WebView2 沙箱中，默认无法访问本地文件系统
- 外部程序启动（`Run` 系列）需显式配置，且不阻断应用加载
- 建议在受控网络环境下使用，避免加载不可信 Web 内容

### 7.2 数据隐私

- 本产品不收集任何用户数据
- 所有配置和应用数据存储在本地
- 日志文件仅记录运行状态，不记录敏感业务数据

## 八、常见问题（FAQ）

### Q1：Web 应用能否访问本地文件？

**A**：默认不支持。如需此能力，可通过后续版本的权限模型进行授权。

### Q2：切换应用时，前一个应用的状态会丢失吗？

**A**：会。当前设计为单应用运行模式，切换后原应用状态销毁。如需保留状态，建议使用 Web 应用自身的持久化机制（localStorage/Cookie）。

### Q3：`Run` 字段是否支持带参数的程序启动？

**A**：支持。可直接填写带参数的命令，如 `"Run": "cmd /c start-local-server.bat --port 8080"`。

### Q4：如何批量部署到多台机器？

**A**：推荐使用 MSI 安装包 + 统一配置文件分发的方案。后续版本将支持注册表配置覆盖。

### Q5：WebView2 运行时如何分发？

**A**：应用启动时会检测 WebView2 运行时，如未安装会引导用户下载。企业环境可提前通过 IT 工具批量安装。
